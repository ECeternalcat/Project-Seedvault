# Project Seedvault 🌱🗃️  
### 为危机中的社区保存生存知识  

[![返回多语言首页](https://img.shields.io/badge/←_返回多语言首页-gray?style=flat)](README.md)

> **“当网络中断，知识依然可及。”**  
> Project Seedvault 是一个**去中心化、开放获取的知识档案库**，专为**无网络、无电力、基础设施损毁**的环境设计。服务于战乱地区、灾区及任何连接中断的社区。

---

## 🌍 包含哪些内容？

我们精选经实地验证的权威资源，打包为可离线使用的模块：

| 模块 | 内容 | 主要来源 |
|------|------|----------|
| **紧急生存包**（<100 MB） | 急救处理、饮用水净化、临时庇护所搭建、危险识别（地雷/未爆弹） | 世界卫生组织（WHO）、Hesperian 基金会《无医可求》、红十字国际委员会（ICRC） |
| **社区生存包**（1–5 GB） | 无电网能源方案、食物生产技术、网状网络通信、心理急救 | 联合国粮农组织（FAO）、Appropedia 开源技术库、红十字会 |
| **重建知识包**（10–30 GB） | 基础教育课程、社区自治指南、开源硬件蓝图、国际人道法 | 联合国教科文组织（UNESCO）、维基媒体基金会、OpenStreetMap |

所有内容均：
- ✅ **非军事、非暴力**（不含任何武器制作内容）
- ✅ **公共领域或开放许可**（CC BY-SA 4.0）
- ✅ **经人道组织实地验证**

## 🛠️ 如何使用？

- **智能手机**：安装 [Kiwix](https://www.kiwix.org/zh/) 阅读维基百科离线版  
- **笔记本电脑**：解压后打开 `index.html` 本地浏览  
- **增强体验**：搭配本地AI模型（如通义千问Qwen2.5-Omni-7B）智能查询知识  

小文件（<50 MB）可直接在仓库 [`/packages`](/packages) 目录获取。

## 🔒 为何采用去中心化分发？

中心化服务器易被封锁或摧毁。通过 **BitTorrent + IPFS + 物理媒介**，我们确保：
- 即使互联网割裂，知识仍可本地共享  
- 社区无需依赖外部援助即可自救  
- 无单点故障（No single point of failure）
