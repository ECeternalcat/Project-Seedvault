# Project Seedvault 🌱🗃️  
### 一个面向人类韧性的离线知识库

[![返回多语言首页](https://img.shields.io/badge/←_返回多语言首页-gray?style=flat)](README.md)

> **“当网络崩溃，知识仍须生长。”**  
> Project Seedvault 是一个**去中心化、开放获取的知识档案库**，专为在**无互联网、无电力、无基础设施**的环境中使用而设计。它服务于冲突地区、灾区或任何连接中断的社区。

---

## 🌍 包含哪些内容？

我们精选了经过实地验证的权威资源，打包为可离线使用的模块：

| 模块 | 内容 | 主要来源 |
|------|------|--------|
| **紧急生存包**（<100 MB） | 急救、净水、庇护所搭建、危险识别（如地雷、化学泄漏） | 世卫组织（WHO）、Hesperian 基金会、红十字国际委员会（ICRC） |
| **社区韧性包**（1–5 GB） | 离网能源、食物生产、Mesh 自组网通信、心理健康 | 联合国粮农组织（FAO）、Appropedia、红十字会 |
| **长期重建包**（10–30 GB） | 基础教育、社区治理、开源硬件、法律与人权 | 联合国教科文组织（UNESCO）、维基媒体、OpenStreetMap |

所有内容均：
- ✅ **非军事、非暴力**
- ✅ **公共领域或开放许可**
- ✅ **经人道组织验证**

## 🛠️ 如何使用？

- **手机**：安装 [Kiwix](https://www.kiwix.org) 阅读 `.zim` 文件  
- **电脑**：解压后打开 `index.html`  
- **搭配 AI**：运行本地模型（如 Qwen2.5-Omni-7B 或 gpt-oss-20b）智能查询知识库  

小于 50 MB 的小文件可直接在本仓库 `/packages` 目录中获取。

## 🔒 为何采用去中心化分发？

中心化服务器可能被封锁、轰炸或审查。通过 **BitTorrent + IPFS + 物理分发**，我们确保：
- 知识在互联网碎片化时依然存活  
- 社区可在本地共享，无需外部依赖  
- 无单点故障
